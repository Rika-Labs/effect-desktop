# Validate Bridge Metadata

## Planned

Issues #453 through #456 targeted bridge metadata that was generated by framework code before host transport: request timestamps, cancel timestamps, generated stream ids, and stream backpressure sizes.

## Shipped

Request timestamp and empty stream id validation were already present in the current branch. This change added coverage for invalid cancel timestamps during abort and tightened method/stream backpressure so `size: 0` fails contract registration. Event backpressure remains unchanged because it is tracked by a separate issue.

## Review Surface

The key boundary is `packages/bridge/src/contracts.ts`: method and stream output backpressure now require positive sizes, while event specs still use the prior non-negative rule. The cancel test proves abort listeners do not throw when the cancel timestamp generator returns `NaN`.

## Lesson

Client-generated protocol metadata needs the same typed boundary as host input. Schema constructors are useful as contracts, but callers should not rely on constructor defects as the runtime error model.
