---
title: Testing
description: Test desktop behavior against ports without launching a desktop app.
---

Application behavior should be testable without real windows or OS prompts.

Ports make that possible. The same Effect program can run against memory-backed services, a mock host protocol, and deterministic event logs.

```ts
test("save writes the note", async () => {
  const result = await Effect.runPromise(
    saveNote(note).pipe(
      Effect.provide(MemoryFilesystem.layer),
      Effect.provide(MockHostProtocol.layer)
    )
  )

  expect(result.path).toBe("/notes/today.md")
})
```

The test proves the app behavior, not the operating system.

## Native services

Native service tests should swap layers instead of constructing bridge envelopes by hand.

`TestScreen.layer()` is the Layer-first proof for native services: the same `Screen` program can run against live, bridge-client, and deterministic test providers by changing layers.

`TestWindow.layer()` follows the supported Window client surface. It records `Window.create` and `Window.close` calls and tracks open handles. Descriptor-only unsupported Window methods are not present on the test client.
