---
title: Desktop config
description: The desktop config is the deployable contract for app identity, runtime entry, and renderer output.
---

`desktop.config.ts` is plain data. The CLI reads it to build, package, sign, notarize, publish, and diagnose the app.

```ts
export default {
  app: {
    id: "dev.acme.notes",
    name: "Acme Notes",
    version: "1.0.0"
  },
  runtime: {
    entry: "src/runtime.ts"
  },
  renderer: {
    dist: "dist"
  }
} as const
```

The config should describe the application. It should not run application logic.
