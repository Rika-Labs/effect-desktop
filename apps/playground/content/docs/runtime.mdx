title: Runtime
description: Long-running work has a public lifecycle.

Processes, PTYs, workers, settings, SQLite, event logs, and jobs get cancellation and devtools visibility by default.

## Pty.spawn

Long-running resources have handles, scope ownership, and observable output.

<Example
  title="Pty.spawn"
  summary="Stream output. Scope owns cleanup."
  accent="amber"
  successOutput="streaming output · handle visible"
  interruptedOutput="process group killed · stream closed"
>

```ts
const terminal = yield* Pty.spawn({
  command: "bun",
  args: ["test", "--watch"],
  cwd: workspace
})

yield* terminal.output.pipe(
  Stream.runForEach(renderLine),
  Effect.scoped
)
```

</Example>

## Job.cancel

Background work has a public lifecycle — not hidden promises.

<Example
  title="Jobs.start / Jobs.cancel"
  summary="Registered, observable, interruptible."
  accent="slate"
  successOutput="job running · progress visible"
  interruptedOutput="job canceled · children interrupted"
>

```ts
const job = yield* Jobs.start("index-workspace", {
  effect: indexWorkspace(root),
  interruptible: true
})

yield* Jobs.cancel(job.id)
```

</Example>
