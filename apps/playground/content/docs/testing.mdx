Because native capabilities are ports, tests run the same Effect program against memory-backed services, mock host protocol clients, and deterministic event logs.

## Test app behavior

Exercise the same save workflow with an in-memory filesystem and a fake host.

<Example
  title="MemoryFilesystem · MockHostProtocol"
  summary="Same effect. Real assertions. No OS dependency."
  accent="emerald"
  successOutput="PASS · /notes/today.md exists"
  interruptedOutput="test fiber interrupted · memory state discarded"
>

```ts
test("save writes the note", async () => {
  const result = await Effect.runPromise(
    saveNote(note).pipe(
      Effect.provide(MemoryFilesystem.layer),
      Effect.provide(MockHostProtocol.layer)
    )
  )

  expect(result.path).toBe("/notes/today.md")
})
```

</Example>

## Test window intent

Assert the renderer asked for the right native operation without opening a real window.

<Example
  title="MockHostProtocol.calls"
  summary="Captured envelopes are deterministic."
  accent="blue"
  successOutput="Window.open · /settings"
  interruptedOutput="pending call cleared · mock host reset"
>

```ts
const host = MockHostProtocol.make()

await Effect.runPromise(
  openSettings.pipe(Effect.provide(host.layer))
)

expect(host.calls).toContainEqual({
  method: "Window.open",
  input: { route: "/settings" }
})
```

</Example>
