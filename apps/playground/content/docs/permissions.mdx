---
title: Permissions
description: Dangerous operations require explicit, auditable approval.
---

Process execution, secrets, filesystem writes, external opens, and updates produce auditable decisions.

## Permission.ask

Dangerous work is explicit, auditable, and recoverable through typed errors.

<Example
  title="Permission.ask"
  summary="Capability + reason → grant token (or typed denial)."
  accent="emerald"
  successOutput="approved · GrantToken issued"
  interruptedOutput="request canceled · no token"
>

```ts
const grant = yield* Permission.ask({
  capability: "process.spawn",
  reason: "Run the project test command"
})

yield* Effect.if(grant.allowed, {
  onTrue: () => Process.spawn(command),
  onFalse: () => Effect.fail(new PermissionDenied())
})
```

</Example>

## Typed denial

Expected failures are values the caller must handle.

<Example
  title="Either(PermissionDenied)"
  summary="No throw. No silent swallow. Audit gets the same value."
  accent="amber"
  successOutput="Left(PermissionDenied) · audit recorded"
  interruptedOutput="effect interrupted · audit skipped"
>

```ts
const result = yield* Effect.either(
  Secrets.read("api-token")
)

if (Either.isLeft(result)) {
  yield* Audit.record(result.left)
}
```

</Example>
