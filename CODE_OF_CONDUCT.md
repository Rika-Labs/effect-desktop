# Code of Conduct

Rika Labs expects contributors to keep the project technically rigorous and
professionally safe.

## Expected Behavior

- Be precise, respectful, and direct.
- Challenge assumptions with evidence.
- Treat security reports and production risks seriously.
- Give reviewers enough context to verify your claims.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Publishing private vulnerability details before a fix is available.
- Misrepresenting test results, production impact, or repository state.
- Repeatedly ignoring maintainer guidance or project boundaries.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues; decline
contributions; or restrict participation when behavior undermines the project.

Report conduct concerns to dallenpyrah@rikalabs.sh.
